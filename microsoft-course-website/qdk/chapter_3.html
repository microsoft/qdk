Chapter 3: Simulation Backends

The QDK ships four simulation backends that trade off exactness, memory, and circuit class. Knowing which to reach for вЂ” and when вЂ” is what separates a workflow that scales from one that runs out of RAM at 30 qubits. This chapter covers the full state vector simulator, the Clifford/stabilizer backend, noisy simulation with built-in noise models, qubit loss modeling, and arbitrary Pauli noise channels. By the end you can select the right backend for any circuit and quantify the effect of noise on measurement fidelity.

What you'll learn

  • Use the full state vector simulator and understand its 2^n memory limit
  • Read DumpMachine() and DumpRegister() output to inspect quantum state at any point
  • Use the Clifford/stabilizer backend for polynomial-time simulation of Clifford-only circuits
  • Apply built-in noise models: DepolarizingNoise, BitFlipNoise, PhaseFlipNoise
  • Model qubit loss and guard against undefined behaviour with IsLossResult
  • Specify arbitrary Pauli noise channels with (px, py, pz) tuples
  • Choose the right backend based on circuit class, qubit count, and noise requirements
