Chapter 2: Q# and the QDK Essentials

Q# is a purpose-built quantum programming language that enforces qubit scoping, reversibility, and type safety at the compiler level. This chapter covers the language fundamentals you will use in every subsequent chapter: the operation/function distinction, qubit allocation, adjoint and controlled functors, the standard library, and the Python interop API. It also covers two critical integration points — OpenQASM 3 and Qiskit — that let you bring existing circuits and hardware workflows into the QDK ecosystem.

What you'll learn

  • Distinguish Q# operations from functions, and use partial application to compose new callables
  • Allocate and scope qubits with use; understand why reset before release is mandatory
  • Mark operations is Adj + Ctl and use within/apply for automatic uncomputation
  • Navigate the standard library: Std.Arrays, Std.Math, Std.Measurement, Std.StatePreparation
  • Call Q# from Python via %%qsharp, qsharp.eval(), qsharp.run(), and qsharp.code.*
  • Organise programs with namespaces and call namespace-qualified operations from Python
  • Run, compile, and import OpenQASM 3 programs; integrate Qiskit circuits via QSharpBackend

2.1 Operations vs Functions

In Q#, functions are purely classical — they compute a value from inputs with no side-effects and cannot touch qubits. Operations can allocate qubits, apply gates, and measure. This separation lets the compiler reason about reversibility and enforce functor constraints at compile time.

Key syntax:

  • operation Name(args) : ReturnType { ... } — can allocate qubits, has side effects
  • function Name(args) : ReturnType { ... } — pure, no qubits, no side effects
  • Partial application: CNOT(control, _) fixes the control qubit and returns a new Qubit => Unit operation — the underscore is a placeholder for the remaining argument

2.2 Qubit Allocation and Scoping

use q = Qubit() allocates a fresh qubit guaranteed to be in |0⟩. For arrays: use qs = Qubit[n]. Qubits are scoped to the use block — they are released automatically when the block exits.

Q# enforces that every qubit is back in |0⟩ before release; the simulator raises an error otherwise. This makes qubit leaks and entanglement bugs impossible to miss.
